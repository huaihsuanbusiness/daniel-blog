> 這篇適合誰？  
> 已經把 ComfyUI 跑起來，接下來想裝模型，但一打開 Civitai 或 Hugging Face 就被各種 checkpoint、LoRA、版本號、下載按鈕淹過去的人。  
> 如果你剛完成安裝，這篇很適合接在上一篇後面看。

本地生圖真正讓人眼花的，通常不是 ComfyUI 本身。

真正會把人淹沒的，是模型下載世界。

你一打開網站，眼前會是這種場面：

- 這個模型寫 SDXL
- 那個寫 Flux
- 另一個又寫 Pony
- 同一頁有 checkpoint、有 LoRA、有 VAE
- 檔名長得像亂碼
- 頁面底下還有十幾張範例圖，每張看起來都很猛

這時候如果你還沒建立判斷，很容易發生兩件事：

1. 下載一堆其實自己根本用不到的檔案
2. 下載了也不一定能用，因為 base model 根本不相容

所以這篇不只是介紹兩個網站而已。我更想幫你先建立一個簡單但夠用的地圖：

> **Civitai 比較像逛模型社群和找風格靈感的地方；Hugging Face 比較像拿官方模型、看版本、看說明、抓原始檔案的地方。**

兩邊都很重要，但用途不一樣。

## 先講一句最實用的

如果你現在是剛入門的新手，我會這樣建議：

- **想知道最近大家都在玩什麼、想看範例圖、想找風格模型**，先去 **Civitai**
- **想拿比較原始、比較官方、比較容易追版本的模型檔**，先去 **Hugging Face**

你不需要選邊站。大部分玩本地模型的人，本來就是兩邊都會用。

## Civitai 是什麼？

Civitai 比較像一個圍繞 Stable Diffusion 生態長出來的模型社群廣場。

它的強項不是只有「提供下載」，而是把模型、範例圖、生成參數、社群互動、熱門程度，全部放在同一個場景裡。你不是只在拿一個檔案而已，你是在看別人怎麼用這個模型、用出什麼風格、大家為什麼喜歡它。

你可以把它想成：

- 有點像模型版的 Pinterest
- 也有點像比較視覺導向的模型市集
- 同時還帶一點作品社群的味道

它很適合做這幾種事：

- 找風格模型
- 看別人用什麼 prompt 和參數
- 判斷某個 checkpoint 大概會出什麼樣子
- 找寫實、動漫、特定畫風、人物 LoRA 這類社群資產

### Civitai 最大的優點，不是模型多而已

它真正迷人的地方是：**你可以先看圖，再決定要不要下載。**

這件事很重要。因為本地模型不是 app store，你不能只靠名字判斷它到底是猛男還是地雷。Civitai 的範例圖、生成資料、留言與人氣，會讓你比較快知道：

- 這個模型偏寫實還是偏插畫
- 它的畫風乾不乾淨
- 它比較吃什麼 base model
- 大家是拿它當主 checkpoint，還是拿它當補味道的 LoRA

## Hugging Face 是什麼？

Hugging Face 就不是這種「先看圖再決定」的地方了。

它更像一個大型開源模型中心。除了圖片模型，還有語言模型、語音模型、資料集、demo、程式套件，整個生態比 Civitai 廣很多。

如果 Civitai 比較像熱鬧的模型市集，Hugging Face 比較像有版本控制、有 model card、有 repo 結構的模型倉庫。

它的幾個關鍵特色是：

- repo 化管理
- 版本比較清楚
- model card 比較完整
- 很多官方或研究團隊直接在這裡釋出
- 你可以看到檔案結構，而不是只看到一顆下載按鈕

這也是為什麼大家常說很多「原始模型」或「官方權重」都會先從 Hugging Face 找起。因為它本來就比較像模型供應鏈的基礎設施，而不是純展示平台。

## 為什麼大家都去 Hugging Face？

這題其實很多新手會問，而且問得很合理。

因為你逛一圈之後會發現，很多教學文最後都會貼出一個 Hugging Face 連結，好像所有模型最後都要回到那裡。

原因大概有四個：

### 1. 很多官方模型或原始釋出就在那裡

對很多開源模型來說，Hugging Face 是最自然的發佈地點。團隊可以把檔案、說明、版本、授權、README 全部放在同一個 repo。

### 2. model card 很重要

Hugging Face 的 model card 通常會告訴你：

- 這是什麼模型
- 它是基於什麼東西訓練的
- 怎麼用
- 有哪些限制
- 需要注意哪些事項

這些東西對新手非常重要，因為它會直接決定你下載下來之後是不是能正常用。

### 3. 它的 repo 思維比較適合追版本

如果你只是想看漂亮範例圖，Civitai 很舒服。  
但如果你想知道「這個檔到底是哪個版本」「還有哪些附帶檔」「README 寫了什麼」，Hugging Face 通常更清楚。

### 4. 整個開源 AI 世界都在那裡交會

很多工具、教學、套件、下載器，本來就是圍著 Hugging Face 生態在轉。你之後會看到很多工具直接支援 HF repo、HF token、HF 檔案結構，這不是巧合，是因為它本來就是模型供應鏈的一條主幹。

## Civitai 跟 Hugging Face，差別到底在哪？

我先給你一張比較不花俏、但比較實用的表。

| 面向 | Civitai | Hugging Face |
| --- | --- | --- |
| 主要感受 | 模型社群、風格探索、看圖找靈感 | 模型倉庫、版本管理、技術文件 |
| 強項 | 範例圖、社群熱門度、生成資料 | model card、repo 結構、官方釋出 |
| 適合找什麼 | checkpoint、LoRA、各種社群風格模型 | 官方基礎模型、文字編碼器、VAE、研究模型、原始權重 |
| 新手最有感的好處 | 先看結果再決定要不要下載 | 比較容易看懂檔案和版本脈絡 |
| 最容易踩的坑 | 被人氣和範例圖沖昏頭，下載一堆不相容模型 | 頁面比較工程向，初看容易不知道自己該抓哪個檔 |

如果要把這張表再濃縮一點，就是：

> **Civitai 幫你回答「我想要什麼風格」；Hugging Face 幫你回答「這個模型原本是怎麼被發佈的」。**

## 模型頁面到底要看什麼？

這段很重要。因為真正會救你的，不是多逛十個網站，而是**學會看一個模型頁面的重點資訊。**

不管是在 Civitai 還是 Hugging Face，我通常會先看這幾個欄位。

### 1. 這到底是什麼類型的檔？

先分清楚它是：

- **Checkpoint**：主模型，通常最大顆
- **LoRA**：附加風格或能力的小補丁
- **VAE**：影像解碼相關元件
- **Text encoder / CLIP / T5XXL**：文字理解相關元件
- **Embedding / Textual Inversion**：比較小型的文字特徵補件

這一步如果沒分清楚，後面幾乎一定會裝錯位置、選錯 Loader。

### 2. 它是基於哪個 base model？

這是新手最容易忽略，但最重要的一格。

你一定要看它到底是：

- SD 1.5 系
- SDXL 系
- Flux 系
- Pony 系
- 或其他分支

因為 **LoRA 跟 checkpoint 不是隨便混搭都行**。你拿 SDXL 的 LoRA 去套 SD 1.5，通常不會有好結果；你拿 Flux 的元件去接傳統 SD workflow，也不會平白無故成功。

### 3. 檔案格式是什麼？

看到這裡，先優先找：

- `.safetensors`

然後對這些格式提高警覺：

- `.ckpt`
- `.pt`
- 某些 pickle 類型檔案

我不是說它們一定有問題，而是你如果有得選，**先選 safetensors 通常比較安心。**

### 4. 版本頁面是不是有講清楚用途？

尤其在 Civitai，很多模型會有好幾版。有些是作者調過幾次，有些是不同訓練階段，有些則只是為了不同用途拆版本。

你要看的是：

- 版本更新在改什麼
- 作者推薦哪一版
- 範例圖是用哪一版生成的
- 這版是不是還需要額外元件

### 5. 範例圖跟生成資料值不值得信

在 Civitai 看範例圖時，我會建議你不要只看「美不美」，還要看：

- 這些圖是不是同一風格洗出來的
- prompt 有沒有公開
- sampler、steps、CFG 這些資訊有沒有給
- 有沒有很明顯靠後製或額外工具硬撐起來

範例圖有用，但不要把它當神諭。它比較像試吃，不是保證書。

## 新手最常踩的坑，其實不是下載不到，而是下載錯

我看過最常見的幾種情況是：

### 坑 1：只看模型名字，不看 base model

「Realistic」、「Turbo」、「XL」、「Flux」這些字看起來都很誘人，但你不能只看名字。**base model 不相容，比你參數調錯還更致命。**

### 坑 2：看到很多人下載，就直接抓

熱門不代表適合你現在這台機器，也不代表適合你這條 workflow。尤其你現在是 Mac mini M4 16GB，真的不用一上來就狂抓超重模型。

### 坑 3：把 Civitai 當成絕對真理

Civitai 很好用，但它終究是社群平台。留言、人氣、範例圖都有參考價值，但不是每個上傳者都會把技術細節寫得很完整。你有時候還是要回頭看 Hugging Face repo、README、或官方說明。

### 坑 4：看到下載按鈕就先按，不看檔案型別

你現在下載的到底是 checkpoint、LoRA、VAE，還是某個額外文字編碼器？這一定要先搞清楚。不然後面放資料夾時就會全亂掉。

## 那我到底該先去哪裡找？

如果你問我「今天要裝第一個模型，我先去哪裡」，我的答案是這樣：

### 想要穩：先找官方或清楚來源

像基礎模型、文字編碼器、某些關鍵元件，我通常會優先找 **Hugging Face 上原始釋出或清楚維護的來源**。

### 想要快點找到喜歡的風格：先逛 Civitai

如果你現在想找寫實風格、動漫風格、某個流行 checkpoint，Civitai 的探索體驗通常比較直覺。

### 最好的做法其實是兩邊交叉看

很常見的情況是：

1. 先在 Civitai 發現某個模型看起來不錯
2. 再回頭確認它的 base model、版本資訊、作者說明
3. 若有更原始或更清楚的來源，再去 Hugging Face 補查

這樣比較不容易被單一頁面的漂亮圖帶著跑。

## 安全這題，真的要有基本警覺

這裡我不打算寫得很恐嚇，但也不想裝作沒事。

### 先講最實際的結論

**有得選時，優先選 `.safetensors`。**

理由很簡單。safetensors 這種格式的設計目標之一，就是降低傳統 pickle 類型權重檔可能帶來的風險。對本地玩家來說，這是最容易做到、也最值得做的基本防線。

### 平台本身有沒有做安全掃描？

Hugging Face 這幾年在 Hub 端有做惡意檔與 pickle 相關掃描，這很好，但它不等於你可以完全放棄判斷。  
Civitai 則更偏社群模型平台，你更需要自己看來源、看說明、看檔案型態。

### 不要下載來源太糊、說明太空、又沒有範例或背景的東西

尤其是你現在還在入門期，真的不需要把自己的模型資料夾變成淘寶盲盒。

## 如果模型頁面很多資訊，我最少要看哪幾個？

如果你很懶，那至少看這五個：

1. **模型類型**
2. **base model**
3. **檔案格式**
4. **版本說明**
5. **範例圖是不是跟你的需求接近**

這五個先看，你踩坑機率就會少很多。

## 我會怎麼建議新手建立自己的下載習慣？

我自己的建議很簡單：

### 第一步：每次只裝少量模型

不要一口氣抓十幾顆。你最後只會搞不清楚哪顆是哪顆。

### 第二步：把來源記下來

你今天下載的是哪個頁面、哪個版本、哪個 base model，記一下。之後真的會救你。

### 第三步：不要把「好看」跟「適合你現在」混在一起

有些模型真的很強，但不一定適合你這台機器，也不一定適合你現在的 workflow。

### 第四步：官方元件跟社群風格模型分開看

基礎模型、文字編碼器、VAE 這些比較像地基。  
風格 checkpoint、LoRA 比較像裝潢。  
先把地基搞清楚，再去瘋裝潢，後面比較不容易亂掉。

## 小結

如果這篇只留一句在你腦子裡，我希望是這句：

> **不要把下載模型當抽卡，把看模型頁面當成工程判斷。**

Civitai 很適合找風格、看範例、逛社群。  
Hugging Face 很適合找原始釋出、看 model card、追版本與抓關鍵元件。  
兩邊一起用，才是最正常也最省坑的路。
